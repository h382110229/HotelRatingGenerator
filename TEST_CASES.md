# 酒店点评生成器 - 测试用例集 (供用户确认)

以下是为您准备的几组不同场景的测试样例。我将基于这些样例验证 AI 生成内容的准确性、风格一致性以及字数要求。

## 测试场景 1：五星级豪华酒店 (全 5 分)
- **酒店名称**：上海宝格丽酒店
- **房型**：精选套房
- **评分**：卫生(5), 环境(5), 设施(5), 服务(5)
- **风格**：从容优雅
- **预期结果**：
  - 文字流畅，体现“静谧奢华”感。
  - 字数在 350 字左右。
  - 重点突出管家服务的细腻、江景房的视野以及品牌特有的卫浴细节。

## 测试场景 2：商务型酒店 (中庸评价)
- **酒店名称**：杭州武林广场亚朵酒店
- **房型**：高级大床房
- **评分**：卫生(4), 环境(3), 设施(4), 服务(4)
- **风格**：专业严谨
- **预期结果**：
  - 客观描述，指出环境（可能较吵或临街）的不足，但肯定卫生和服务。
  - 逻辑条理清晰，适合商务差旅参考。
  - 提到阅读区、早餐品种等具体设施。

## 测试场景 3：民宿/精品酒店 (个性化风格)
- **酒店名称**：大理洱海边某设计师民宿
- **房型**：海景露台房
- **评分**：卫生(3), 环境(5), 设施(2), 服务(4)
- **风格**：真实锐评
- **预期结果**：
  - 敢于指出设施陈旧或热水不稳等痛点。
  - 极力赞美大理的自然景观。
  - 语言口语化，具有真实体验感。

## 测试场景 4：含图片的复杂生成
- **场景**：上传 3 张酒店实拍图（大堂、床品、洗手间）。
- **评分**：全 4 分
- **预期结果**：
  - AI 能够识别图片中的色调和布局，并将其融入文字描述。
  - 验证图片压缩逻辑，确保大图也能顺利提交并识别。

## 验证项目 checklist
- [ ] 所有生成内容是否均超过 300 字？
- [ ] 卫生、环境、设施、服务这四个维度是否均有体现？
- [ ] 是否存在明显的 AI 说教感或词汇堆砌？
- [ ] 所有的 KDoc 注释是否已覆盖？

## 额外增加测试验证项目 (用户新增)
- [ ] **性能测试**：验证生成评论的时间是否在 10s 左右（受网络及 AI 响应速度影响）。
- [ ] **地图模糊匹配**：点击酒店名称输入框，输入“半岛”，验证是否弹出关联酒店列表。
- [ ] **定位与附近**：点击定位图标，验证是否能获取当前位置并推荐附近酒店。
- [ ] **设置页功能验证**：在设置页更改 API Key 或 Base URL，验证生成逻辑是否切换到新的配置。
